# 附錄 B - AI 代理式互動：從 GUI 到真實世界環境

AI 代理（agent）正越來越常透過與數位介面和實體世界互動來執行複雜任務。它們在這些多樣環境中感知、處理並採取行動的能力，正在從根本上改變自動化、人機互動與智慧系統。本附錄探討代理如何與電腦及其環境互動，並強調相關進展與專案。

## 互動：代理與電腦

AI 從對話夥伴演進為主動、任務導向的代理，正是由 Agent-Computer Interfaces（ACIs）所推動。這些介面讓 AI 能直接與電腦的 Graphical User Interface (GUI) 互動，使其能像人類一樣感知並操作圖示和按鈕等視覺元素。這種新方法超越了傳統自動化中依賴 API 與 system call、僵硬且仰賴開發者的腳本。透過使用軟體的視覺「前門」，AI 現在能以更具彈性且更強大的方式自動化複雜的數位任務，這個過程包含幾個關鍵階段：

* **視覺感知：** 代理首先擷取螢幕的視覺表示，本質上就是截取螢幕畫面。  
* **GUI 元素辨識：** 接著，它會分析這張影像，以區分各種 GUI 元素。它必須學會「看見」螢幕，不只是把螢幕視為像素的集合，而是把它理解為具有互動元件的結構化版面，能分辨可點擊的「Submit」按鈕與靜態橫幅影像，或可編輯文字欄位與單純標籤。  
* **上下文詮釋：** ACI 模組作為視覺資料與代理核心智慧（通常是大型語言模型（large language model, LLM））之間的橋梁，會在任務上下文中詮釋這些元素。它理解放大鏡圖示通常代表「搜尋」，或一組 radio button 代表一個選項。此模組對強化 LLM 的推理至關重要，讓它能根據視覺證據形成計畫。  
* **動態行動與回應：** 然後，代理會以程式方式控制滑鼠與鍵盤來執行計畫，包括點擊、輸入、捲動與拖曳。關鍵在於，它必須持續監控螢幕上的視覺回饋，動態回應變更、載入畫面、彈出式通知或錯誤，才能成功瀏覽多步驟工作流程。

這項技術已不再只是理論。幾個領先的 AI 實驗室已開發出可運作的代理，展現 GUI 互動的力量：

**ChatGPT Operator (OpenAI)：** ChatGPT Operator 被構想為數位夥伴，設計目標是在桌面上直接跨多種應用程式自動化任務。它能理解螢幕上的元素，因此可以執行各種動作，例如把資料從試算表轉移到 客戶關係管理（customer relationship management, CRM）平台、在航空公司與飯店網站之間預訂複雜旅遊行程，或填寫詳細的線上表單，而不需要為每項服務取得專門的 API 存取權。這讓它成為一種可普遍適應的工具，目標是透過接手重複性的數位雜務，提升個人與企業生產力。

**Google Project Mariner：** Project Mariner 是一個研究原型，作為 Chrome 瀏覽器中的代理運作（見 Fig. 1）。其目的是理解使用者意圖，並代表使用者自主執行網頁任務。例如，使用者可以要求它在特定預算與社區範圍內尋找三間出租公寓；Mariner 接著會前往房地產網站、套用篩選條件、瀏覽清單，並將相關資訊擷取到文件中。此專案代表 Google 對真正有幫助且「代理式」網頁體驗的探索，在這種體驗中，瀏覽器會主動為使用者工作。

**Anthropic's Computer Use：** 這項功能讓 Anthropic 的 AI 模型 Claude 成為電腦桌面環境的直接使用者。透過擷取螢幕截圖來感知畫面，並以程式方式控制滑鼠與鍵盤，Claude 可以協調跨越多個彼此未連結應用程式的工作流程。使用者可以要求它分析 PDF 報告中的資料、開啟試算表應用程式對該資料進行計算、產生圖表，然後把圖表貼到電子郵件草稿中；這一連串任務過去需要人類持續輸入才能完成。

**Browser Use**：這是一個開放原始碼程式庫，提供用於程式化瀏覽器自動化的高階 API。它讓 AI 代理能透過取得並控制 Document Object Model (DOM)，與網頁進行介接。該 API 將瀏覽器控制協定中複雜的低階命令抽象化，轉換為更簡化且直覺的一組函式。這讓代理能執行複雜的動作序列，包括從巢狀元素擷取資料、提交表單，以及在多個頁面之間自動導覽。因此，這個程式庫促進了將非結構化網頁資料轉換為結構化格式的過程，使 AI 代理能有系統地處理並用於分析或決策。

## 互動：代理與環境

在電腦螢幕之外，AI 代理也越來越常被設計為能與複雜、動態的環境互動，而這些環境通常映照真實世界。這需要精密的感知、推理與致動能力。

Google 的 **Project Astra** 是推動代理與環境互動邊界的一項代表性計畫。Astra 旨在建立一個能在日常生活中提供協助的通用 AI 代理，運用多模態輸入（視覺、聲音、語音）與輸出，在上下文中理解並與世界互動。此專案聚焦於快速理解、推理與回應，讓代理能透過攝影機與麥克風「看見」並「聽見」周遭環境，在提供即時協助的同時進行自然對話。Astra 的願景是打造一個能無縫協助使用者完成各種任務的代理，範圍從尋找遺失物品到偵錯程式碼，都是透過理解它所觀察到的環境來完成。這超越了單純的語音命令，邁向真正具身地理解使用者當下實體上下文的能力。

Google 的 **Gemini Live** 將標準 AI 互動轉變為流暢且動態的對話。使用者可以對 AI 說話，並以自然的聲音在極短延遲下收到回應，甚至可以在句子中途打斷或切換主題，促使 AI 立即調整。其介面不只限於語音，也允許使用者透過手機相機、分享螢幕或上傳檔案來納入視覺資訊，進行更具上下文感知的討論。更進階的版本甚至能感知使用者的語調，並智慧地濾除無關背景噪音，以便更好地理解對話。這些能力結合後，可以創造豐富的互動，例如只要把相機對準某項任務，就能接收即時操作指示。

OpenAI 的 **GPT-4o model** 是另一種為「omni」互動而設計的選項，表示它能跨語音、視覺與文字進行推理。它以低延遲處理這些輸入，反映人類回應時間，因此能進行即時對話。例如，使用者可以向 AI 展示即時影片串流，詢問正在發生什麼事，或將它用於語言翻譯。OpenAI 為開發者提供「Realtime API」，用來建置需要低延遲、語音對語音互動的應用程式。

OpenAI 的 **ChatGPT Agent** 相較於前代產品，代表了重大的架構進展，具備由新能力組成的整合式框架。其設計納入幾種關鍵功能型態：能自主瀏覽即時網際網路以擷取即時資料；能動態產生並執行運算程式碼，以完成資料分析等任務；也能直接與第三方軟體應用程式介接。這些功能的整合讓代理能根據單一使用者指令，協調並完成複雜的序列式工作流程。因此，它可以自主管理整個流程，例如進行市場分析並產生對應簡報，或規劃物流安排並執行必要交易。在推出的同時，OpenAI 也主動處理這類系統固有的新興安全考量。隨附的「System Card」描述了能在線上執行動作的 AI 所涉及的潛在操作危害，並承認新的濫用途徑。為降低這些風險，該代理的架構包含工程化防護措施，例如針對某些類別的動作要求使用者明確授權，並部署強健的內容過濾機制。該公司目前正與初始使用者群互動，透過以回饋驅動的反覆流程進一步精進這些安全協定。

**Seeing AI,** 是 Microsoft 提供的免費行動應用程式，透過即時旁白描述周遭環境，賦能盲人或低視能人士。此應用程式透過裝置相機運用人工智慧，辨識並描述各種元素，包括物件、文字，甚至人物。其核心功能涵蓋閱讀文件、辨識貨幣、透過條碼辨識產品，以及描述場景與顏色。透過提供更充分的視覺資訊存取，Seeing AI 最終促進視覺障礙使用者獲得更高的獨立性。

**Anthropic's Claude 4 Series** Anthropic 的 Claude 4 是另一種具備進階推理與分析能力的選項。雖然 Claude 4 過去主要聚焦於文字，但它包含強健的視覺能力，可處理來自影像、圖表與文件的資訊。該模型適合處理複雜的多步驟任務，並提供詳細分析。與其他模型相比，即時對話面向並非它的主要焦點，但其底層智慧是為建構高度能幹的 AI 代理而設計。

## Vibe Coding：用 AI 進行直覺式開發

除了直接與 GUI 和實體世界互動之外，開發者使用 AI 建置軟體的方式也正在出現一種新典範：「vibe coding」。這種方法不再依賴精確、逐步的指令，而是依靠開發者與 AI 程式碼助理之間更直覺、對話式且反覆的互動。開發者提供高階目標、期望的「vibe」，或大致方向，AI 則產生符合該方向的程式碼。

此流程具有以下特徵：

* **對話式提示：** 開發者不必撰寫詳細規格，而可能會說：「為一個新應用程式建立簡單、現代感的 landing page」，或「重構這個函式，讓它更符合 Pythonic 且更易讀。」AI 會詮釋「modern」或「Pythonic」這種「vibe」，並產生對應的程式碼。  
* **反覆精煉：** AI 的初始輸出通常只是起點。接著，開發者以自然語言提供回饋，例如：「這是個不錯的開始，但可以把按鈕改成藍色嗎？」或「幫它加上一些錯誤處理。」這種來回互動會持續到程式碼符合開發者期待為止。  
* **創意夥伴關係：** 在 vibe coding 中，AI 扮演創意夥伴，提出開發者可能未曾考慮的想法與解決方案。這能加速開發流程，並帶來更具創新的成果。  
* **聚焦於「做什麼」而非「怎麼做」：** 開發者專注於期望成果（「做什麼」），並把實作細節（「怎麼做」）交給 AI。這讓快速原型設計與探索不同方法成為可能，而不會陷入樣板程式碼的細節。  
* **選用記憶庫：** 為了在較長的互動中維持上下文，開發者可以使用「memory banks」儲存關鍵資訊、偏好或限制。例如，開發者可能會把特定程式碼風格或一組專案需求儲存到 AI 的記憶中，確保未來產生的程式碼能持續符合已建立的「vibe」，而不需要重複說明指令。

隨著 GPT-4、Claude、Gemini 等強大 AI 模型整合到開發環境中，vibe coding 正變得越來越受歡迎。這些工具不只是自動補全程式碼；它們正主動參與軟體開發的創造過程，讓開發更容易上手且更有效率。這種新的工作方式正在改變軟體工程的本質，強調創造力與高階思考，而不是死記語法與 API。

## 重點摘要

* AI 代理正在從簡單自動化，演進為能像人類一樣透過 GUI 視覺化控制軟體。  
* 下一個前沿是真實世界互動，像 Google 的 Astra 這類專案會使用攝影機與麥克風來看見、聽見並理解實體周遭環境。  
* 領先科技公司正在融合這些數位與實體能力，打造能在兩個領域中無縫運作的通用 AI 助理。  
* 這項轉變正在創造一類全新的主動式、具上下文感知能力的 AI 夥伴，能在使用者日常生活中協助完成各式各樣的任務。

## 結論

代理正經歷重大轉型，從基本自動化邁向與數位和實體環境的精密互動。透過運用視覺感知來操作 Graphical User Interfaces，這些代理現在能像人類一樣操作軟體，繞過對傳統 API 的需求。主要科技實驗室正開創這個領域，打造能直接在使用者桌面上自動化複雜、多應用程式工作流程的代理。與此同時，下一個前沿正在擴展到實體世界，像 Google 的 Project Astra 這類計畫使用攝影機與麥克風，在上下文中與周遭環境互動。這些進階系統被設計為具備多模態、即時理解能力，能反映人類互動方式。

最終願景是融合這些數位與實體能力，建立能在使用者所有環境中無縫運作的通用 AI 助理。這項演進也透過「vibe coding」重塑軟體創作本身，形成開發者與 AI 之間更直覺、對話式的夥伴關係。這種新方法優先考量高階目標與創意意圖，讓開發者專注於期望成果，而非實作細節。這項轉變透過把 AI 視為創意夥伴，加速開發並促進創新。最終，這些進展正為一個新的時代鋪路，在這個時代中，主動式、具上下文感知能力的 AI 夥伴能協助我們完成日常生活中的大量任務。
