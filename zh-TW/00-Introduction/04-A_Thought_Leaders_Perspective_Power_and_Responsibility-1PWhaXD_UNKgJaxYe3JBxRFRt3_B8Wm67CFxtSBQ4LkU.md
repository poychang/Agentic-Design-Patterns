# 思想領袖觀點：權力與責任

在過去四十年，我見證過許多科技週期——從個人電腦與 Web 的誕生，到行動與雲端革命——但沒有任何一個週期像這次一樣。多年來，圍繞人工智慧的論述一直有著熟悉的節奏：熱潮與幻滅交替，也就是所謂的「AI 夏天」之後，接著漫長而寒冷的冬天。但這一次，有些事情不同了。對話已明顯轉向。如果過去十八個月談的是引擎——大型語言模型（large language model, LLM）令人屏息、近乎垂直的崛起——那麼下一個時代要談的，將是我們圍繞它打造的車。它將關乎那些能駕馭這股原始力量的框架，將其從產生貌似合理文字的工具，轉化為真正能採取行動的代理。

我承認，一開始我是懷疑的。我發現，貌似合理往往與一個人對某個主題的知識成反比。早期模型不論多麼流暢，都像是帶著某種冒牌者症候群在運作，優先追求可信，而非正確。但接著轉折點來了，一類新的「推理」模型帶來了階躍式變化。突然之間，我們不再只是與一台預測序列中下一個字的統計機器對話；我們開始窺見某種萌芽中的認知形式。

我第一次試用新的代理式 coding 工具時，感受到那種熟悉的魔法火花。我交給它一個自己一直抽不出時間做的個人專案：把一個慈善網站從簡單的網站建置工具，遷移到合適且現代化的 CI/CD 環境。接下來二十分鐘，它開始工作，提出釐清問題、要求憑證，並提供狀態更新。那感覺不像是在使用工具，更像是在與一位初階開發者協作。當它交給我一個完整可部署的套件，還附上無可挑剔的文件與單元測試時，我震撼不已。

當然，它並不完美。它會犯錯，會卡住。它需要我的監督，更重要的是，需要我的判斷來引導它回到正軌。這段經驗讓我深刻體會到一個在漫長職涯中以艱難方式學到的教訓：你不能盲目信任。然而，這個過程非常迷人。窺視它的「思維鏈」就像看著一個心智在運作——混亂、非線性，充滿開始、停頓與自我修正，與我們人類的推理並無太大不同。它不是一條直線；它是朝向解法的隨機漫步。這裡出現了某種新事物的核心：不只是能產生內容的智慧，而是能產生*計畫*的智慧。

這正是代理式框架的承諾。它就像靜態地鐵圖與能即時重新規劃路線的動態 GPS 之間的差異。傳統以規則為基礎的自動機會遵循固定路徑；一旦遇到意外障礙，就會失效。由推理模型驅動的 AI 代理，則有潛力觀察、適應並找到另一條路。它具備某種形式的數位常識，使其能穿越現實世界中無數邊界案例。這代表一種轉變：從單純告訴電腦*要做什麼*，轉向解釋*為什麼*我們需要完成某件事，並信任它找出*如何*完成。

這個新前沿令人振奮，但也帶來深刻的責任感，尤其是從我作為全球金融機構 CIO 的角度來看。利害關係高到難以衡量。代理在創作「雞肉鮭魚融合派」食譜時犯錯，可以是一則有趣軼事；但代理在執行交易、管理風險或處理客戶資料時犯錯，就是實際問題。我讀過那些免責聲明與警世故事：某個 Web 自動化代理在登入失敗後，決定寄信給國會議員抱怨登入牆。這是帶著黑色幽默的提醒：我們正在處理一項尚未完全理解的技術。

此時，工藝、文化，以及對原則毫不鬆懈的專注，便成為我們不可或缺的指引。我們的 Engineering Tenets 不只是紙上的文字；它們是我們的指南針。我們必須*帶著目的建構*，確保每一個設計出的代理，都從清楚理解我們正在解決的客戶問題開始。我們必須*預見轉角之後的風險*，預期失敗模式，並設計出本質上具備韌性的系統。最重要的是，我們必須*啟發信任*，對我們的方法保持透明，並對結果負責。

在代理式世界中，這些準則變得更加迫切。殘酷的事實是，你不能只是把這些強大的新工具覆蓋在混亂且不一致的系統上，然後期待好結果。混亂系統加上代理，就是災難的配方。以「垃圾」資料訓練的 AI，不只是產出垃圾；它會產出貌似合理、充滿自信的垃圾，進而污染整個流程。因此，我們首要且最關鍵的任務，是先整備地基。我們必須投資於乾淨資料、一致的 metadata，以及定義良好的 API。我們必須打造現代化的「州際公路系統」，讓這些代理能安全且高速地運作。這是建構可程式化企業、「enterprise as software」的艱難基礎工作；在其中，我們的流程就像程式碼一樣擁有良好架構。

說到底，這趟旅程不是為了取代人類巧思，而是為了增強它。它要求我們每個人具備一組新技能：清楚解釋任務的能力、懂得委派的智慧，以及驗證輸出品質的勤勉。它要求我們保持謙卑，承認自己不知道的事，並永不停止學習。本書接下來的篇章，將為建構這些新框架提供技術地圖。我希望你使用它們時，不只是建構可能之物，而是建構正確、穩健且負責任之物。

世界正在要求每一位工程師挺身而出。我相信我們已準備好迎接挑戰。

享受這段旅程。

Marco Argenti, CIO, Goldman Sachs
